1. Product Overview 
GoRide Bangladesh is a bilingual car rental and transport marketplace serving individuals, corporate 
clients, car owners, and drivers across Bangladesh. The platform is designed to be mobile-first, Bangla
friendly, and built for scale from day one. 
Attribute

1.1 Business Lines 
• Owners / Drivers — onboard cars and drivers to supply the fleet 
• Corporate Clients — manage company transport, billing, and fleet usage 
• One-Time / Solo Clients — book transport on demand without full registration 
• Tours & Travels — structured tour packages (Phase 2, structure prepared now) 
1.2 Business Objectives 
• Onboard car owners and drivers to build supply 
• Capture one-time and corporate client demand 
• Enable corporate clients to monitor usage and billing independently 
• Give owners/drivers full visibility of trips, ratings, and documents 
• Give admin complete operational control with automation support 
• Build a platform architected for future scaling (app, payments, tracking)

3. User Types & Personas 
3.1 One-Time / Solo Client 
A person who needs a single trip — airport transfer, event, day trip. May be tech-savvy or not. Primarily 
mobile. Wants speed and simplicity with no commitment to register. 
 
Attribute Detail 
Authentication OTP verification (at time of form submission, not 
before) 
Key Actions Fill booking form → OTP verify → submission 
confirmed 
Pain Point Addressed OTP moved to end of funnel; user fills form freely 
first 
 
3.2 Corporate Client 
A company transport manager or admin who needs ongoing fleet management, billing transparency, 
and the ability to submit new car requests without calling the GoRide team. 
 
Attribute Detail 
Authentication Login + Password (email or phone) 
Key Actions View fleet, track billing, submit car requests, view 
history 
Unique Need Billing visibility — outstanding amounts, next 
billing date, invoice download 
 
3.3 Car Owner / Driver 
An individual who owns one or more cars, or drives for an owner. May have limited digital literacy. 
Likely registers via mobile from the field (e.g. sitting in traffic after seeing a Facebook ad). 
 
Attribute Detail 
Authentication OTP for mobile verify, then Login + Password 
Key Actions Register (progressive), upload documents, 
manage cars, view trips/ratings 
Pain Point Addressed Registration split into quick signup (30 sec) + 
dashboard document completion 
Admin Follow-Up If profile incomplete after 3 days, admin can call 
the captured lead

4. UX Flow — Key Improvements (v3.0) 
This section captures the critical UX changes recommended in v3.0. These are not optional — they 
directly impact conversion rates and platform scalability. 
 
4.1 One-Time Client Booking Flow (Revised) 
Previous flow required OTP verification BEFORE the booking form. This creates high drop-off. The 
revised flow moves OTP to the end of the funnel. 
 
Step Action Rationale 
Step 1 User fills full booking form 
(destination, date, car type, etc.)

builds psychological investment

'Submit'

Spam/bot protection is handled via backend rate limiting and CAPTCHA — NOT by front-loading OTP 
before the form.

10. Security Requirements 
All security requirements are mandatory from Phase 1. Security is not a future feature. 
 
Requirement Detail 
HTTPS / SSL All pages served over HTTPS; HTTP redirects to 
HTTPS automatically 
CSRF Protection CSRF tokens on all form submissions 
OTP Rate Limiting Maximum 3 OTP requests per mobile number per 
10 minutes; IP-level limiting also applied 
SQL Injection / XSS All inputs sanitized; parameterized queries; output 
escaping throughout 
Password Policy Minimum 8 characters; bcrypt hashing; no 
plaintext storage 
Role-Based Access Control (RBAC) Admin, Corporate, Owner/Driver, One-Time Client 
— each role sees only permitted data and actions 
File Upload Restrictions Allowed types: JPG, PNG, PDF only; max file size 
enforced; files stored with randomized names 
Session Management Secure session tokens; session timeout after 
inactivity (configurable per role) 
Admin Access Admin panel at non-obvious URL; additional 2FA 
recommended for admin accounts 
Data Privacy Personal data (NID, photos, documents) 
accessible to admin only; not exposed to other 
users

11. SEO Requirements 
SEO is essential for a service business to be discoverable in Bangladesh. All items below are required 
from Phase 1. 
• Unique meta title and description for every page 
• Structured data / schema markup: LocalBusiness, Service, FAQPage where applicable 
• XML Sitemap auto-generated and submitted to Google Search Console 
• robots.txt configured correctly (admin panel excluded from indexing) 
• Bilingual pages use hreflang tags (en / bn) 
• Canonical URLs set on all pages 
• Open Graph tags on all public pages (for Facebook and WhatsApp link preview: title, 
description, image) 
• Google Analytics 4 installed and configured from launch 
• Facebook Pixel installed for ad tracking and retargeting 
• Page speed targets: under 3 seconds on 4G mobile (Bangladesh average conditions) 
• Image optimization: WebP format, lazy loading for sliders and car photos 
• Core Web Vitals: LCP, CLS, FID targets met before launch

14. Performance & Mobile Requirements 
Bangladesh users predominantly access services on mobile with variable connection quality. 
Performance is a product requirement, not an afterthought. 
• Target page load time: under 3 seconds on 4G (Bangladesh average) 
• All images optimized and served in WebP format 
• Lazy loading on sliders, car photo galleries, and logo sections 
• OTP flow fully optimized for mobile — large input, numeric keyboard triggered automatically 
• All forms fully usable on screens as small as 360px wide 
• Dashboards remain functional and readable on mobile screens 
• Language toggle visible and accessible on mobile without extra navigation 
• WhatsApp floating button on all pages — fixed bottom-right corner 
• Browser compatibility: Chrome (Android), Samsung Internet, Firefox, Safari (iOS), Chrome 
(Desktop)
